## 0. The verdict on scope (read this first)

**One web app, one laptop, no mobile app: correct.** Every integration cut is an hour saved and a failure mode removed.

**But "one simple page with camera + analysis" is not enough to win.** A live webcam dashboard alone is a thing DO judges have seen a hundred times, and it forfeits both target prizes. The fix costs nothing extra:

- **One page, THREE states**: `CONSENT → LIVE → DEBRIEF`. Same route, two click-transitions. The `LIVE → DEBRIEF` transition — "the conversation you just had, replayed as an annotated emotional timeline with an AI debrief" — is the entire Best Use of Data prize and the emotional payoff of the demo.
- **The "discreet nudge" contradiction solves itself with staging, not code** (see §6). The laptop screen faces the user only; the partner across the table can't see it. The nudge really is private. The audience sees it via the projector.
- **"Simple" must not mean "one LLM prompt."** Judging explicitly punishes that. The local signal pipeline below (MediaPipe → derived signals → deterministic event engine → LLM only at the edges) is simultaneously the technical-complexity story, the privacy story, the low-latency path, AND the cheapest thing to build. This is not a trade-off; it's a freebie.

## 1. What it is (3 states, 1 page)

### State 1 — CONSENT (build: 30 min, do not skip — it's the ethics moat)
- Partner-facing consent card: exactly what is tracked ("facial expression signals, attention, speech text — video never leaves this laptop"), a live "what the AI sees" mirror preview, a big red kill switch, "session data deleted by default."
- User enters 1 line of context: who they're talking to + what they hope for ("catching up with a friend; I want to know if I talk too much").
- Both click **I consent** → LIVE.

### State 2 — LIVE (the core loop)
- Partner's webcam feed (small), surrounded by the live dashboard:
  - **Engagement line** (the hero signal, updating ~1 Hz)
  - Sparklines: attention/gaze, expressivity, valence, talk-time balance bar (you vs them)
  - Everything shows a **confidence badge** — every single read, always
- **Nudges**: rare (cooldown ≥ 90 s), hedged, with evidence:
  > *"They may be drifting — maybe ask a question?"* · **72%** · gaze away 9s · smile fading · you've held the floor 84s
- **End session** button → DEBRIEF.

### State 3 — DEBRIEF (where the prizes live)
- **Hero chart**: area chart of engagement over the whole conversation; gradient hue shifts with valence (amber = engaged → violet = drifting); pinned event markers (nudge fired, smile peak, attention dip); click a marker to scrub to that moment (transcript snippet shown).
- **AI debrief** streaming in with a typewriter effect, visibly labeled **"generated by Claude on DigitalOcean Gradient"**: what went well, 2–3 timestamped notable moments, one or two kind suggestions, all hedged.
- Sidebar: past sessions (pre-seed 2–3 fake ones so the app doesn't look empty).

## 2. Architecture

```
┌────────────────────────── Browser (Chrome, the demo laptop) ──────────────────────────┐
│  getUserMedia ─→ MediaPipe Face Landmarker (@mediapipe/tasks-vision@0.10.35)          │
│     GPU delegate, VIDEO mode, blendshapes + transformation matrix, throttled ~12 Hz   │
│         ↓                                                                             │
│  Derived-signal layer (pure JS fn): smile, browRaise/Furrow, eyeOpenness, gazeAway,   │
│     nod, lean-in, talking — EMA-smoothed, per-person baseline from first 60–90 s,     │
│     all reads = deltas vs THAT person's own baseline                                  │
│         ↓                                                                             │
│  1 Hz "engagement frame" {t, engagement, valence, attention, signals, confidence}     │
│     → in-memory array (powers live charts + debrief payload + timeline replay)        │
│         ↓                                                                             │
│  Deterministic event engine: EWMA baseline + sustained z-score (e.g. z < −1.5 for     │
│     ≥10 s) + hysteresis + 90–120 s cooldown  ─→ candidate nudge event                 │
│                                                                                       │
│  Web Speech API (webkitSpeechRecognition, continuous, restart in onend) → transcript  │
│  WebAudio RMS/VAD ⊕ partner jawOpen ─→ talk-time attribution (USER vs PARTNER)        │
└────────────┬──────────────────────────────────────────────────────────────────────────┘
             │ fetch (plain HTTPS — no WebSockets needed)
┌────────────▼───────────── DO App Platform: one Node/Express service, port 8080 ───────┐
│  serves built frontend  +  holds the DO inference key                                 │
│  POST /api/nudge   → Gradient fast model, forced tool-call → {text, confidence, ev}   │
│  POST /api/debrief → Gradient Claude, SSE stream → debrief + timeline annotations     │
│  POST /api/frames  → batch insert (every 5 s) into Managed Postgres                   │
└────────────┬───────────────────────────────┬──────────────────────────────────────────┘
             ▼                               ▼
   DO Gradient AI Inference          DO Managed Postgres
   inference.do-ai.run/v1/           sessions + frames tables
   (2+ models — see §3)              (pgvector = stretch)
```

**Privacy by architecture** (say this sentence to the judges): raw video and audio never leave the laptop — only derived 1 Hz signal features and transcript text reach the DO-hosted backend.

**Stack**: Vite + React + TS + Tailwind + shadcn/ui + Recharts + Framer Motion; Node 20 + Express + `pg`. One repo, server serves `dist/`. No WebSockets, no auth, no router beyond the 3 states.

## 4. The signal layer (the "not one prompt" evidence)

Pure function per frame, each output EMA-smoothed with an explicit confidence:

- `smile` = mean(mouthSmileL, mouthSmileR); `browRaise` = browInnerUp + mean(browOuterUp L/R); `browFurrow` = mean(browDown L/R)
- `eyeOpenness` = 1 − mean(eyeBlink L/R)
- `gazeAway` = |head yaw| > ~20° or |pitch| > ~15° sustained (from the 4×4 transform matrix → three.js `Euler.setFromRotationMatrix(m,'YXZ')`), plus eyeLook* deltas
- `nod` = oscillation detection on windowed pitch derivative; `lean` = interocular distance vs rolling baseline; `talking` = rolling variance of jawOpen
- `engagement` = weighted blend of gaze-on-face, nods, smile, lean, expressivity-delta **vs the person's own first-90-seconds baseline** (this per-person baseline is your answer to "faces vary across people and cultures")
- `confidence` (computed **in code**, never asked from the LLM) = face-tracking quality × count of agreeing signals × baseline maturity → low/med/high tier passed INTO the prompt

Nudge trigger example (deterministic + explainable): *engagement z-score < −1.5 sustained 10 s AND user has held the floor > 75 s* → candidate event → LLM phrases it within the given confidence tier, with a hedging lexicon and a ban-list (never "bored", "angry", "upset", no emotion nouns, no diagnoses).

**Never render emotion-category labels anywhere in the UI.** Signal descriptors + hedged suggestions + confidence only. This single decision is the entire scientific/ethical defense (it is literally what the Barrett 2019 critique recommends building instead).

## 6. Demo script (~2 min) + staging

**Staging (this is the trick — decide it now):** Partner (B) sits across the table **facing the laptop camera**. User (A) sits behind the laptop, **screen facing A only**, talking to B face-to-face. B physically cannot see the screen → the nudge is genuinely discreet. Projector mirrors A's screen so the audience gets x-ray vision. Zero extra hardware, and it dissolves the "isn't this a surveillance dashboard?" objection on sight.

1. **0:00–0:20 — hook, told personally:** "Most people get a superpower for free: they can *feel* a conversation going wrong. Millions of neurodivergent people don't get that signal — and it costs them jobs, friendships, first impressions. We built the missing translator."
2. **0:20–0:30 — consent beat:** B taps *I consent* on the mirror view; kill switch visible. Say: **"The video never leaves this laptop — only the understanding runs in the cloud, on DigitalOcean."**
3. **0:30–1:30 — the skit:** A talks (slightly too long) about something dry. B disengages on cue — glances away, checks phone, face goes flat. **The engagement line falls live in front of the judges.** Nudge fires: hedged text + 72% badge + evidence line. A reads it, asks B a question about themselves. B lights up; the line recovers. *That's the wow, inside 90 seconds.*
4. **1:30–2:00 — the payoff:** Click **End**. The conversation the judges just watched replays as the annotated timeline; Claude's debrief streams in, labeled "DigitalOcean Gradient." Close: **"Copilot, never autopilot. It suggests, never diagnoses — and every read shows its confidence."**
5. **Encore (only if the room is warm):** "Want to try to bore my cofounder?" → 30 s with a judge.
6. **Fallback:** rehearsal-recorded backup video one keypress away. If venue wifi dies mid-demo, the app auto-falls back to canned nudge text + cached debrief (everything else is local).

**Ethics Q&A one-liners (memorize):**
- *Surveillance?* → "Runs only with both people's explicit, visible consent — and raw video never leaves the laptop."
- *Emotion AI is pseudoscience?* → "We agree — that's why it never labels emotions. It describes observable signal *trajectories* against the person's *own* baseline, with confidence attached. That's what the science says to build."
- *Fixing neurodivergent people?* → "It's a translator, not a trainer — an accommodation, like captions, and it helps both sides understand each other."
- *Positioning:* "The first live, consent-first build of the two-way translator autistic co-designers specified in CHI 2024 research — aimed at the gap Stanford's Superpower Glass left: context and what-to-do, not emotion flashcards."

## 7. Risk register

| Risk | Mitigation |
|---|---|
| Venue lighting breaks face tracking | Test in the demo room early; MediaPipe is robust; keep a hidden dev panel to tune thresholds |
| Venue wifi dies | Only the 2 LLM calls need network; auto-fallback to canned nudges + cached debrief; backup video |
| Web Speech misbehaves / non-Chrome | Demo in real Chrome only (not Brave/Arc); transcript is an enhancement — debrief degrades gracefully to signals-only |
| DO model names / tool-calling differ from research | Verify `GET /v1/models` + one forced tool call in the first 30 min; keep a fallback model constant |
| Scope creep, no rehearsal | Demo script written at hour 0 is the contract; hard freeze 6:30; two timed rehearsals are non-negotiable |
